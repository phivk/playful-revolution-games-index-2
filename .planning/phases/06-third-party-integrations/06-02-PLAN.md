<objective>
Build the Instagram API infrastructure: server-side Route Handler proxy that fetches recent posts from the Instagram Graph API, a token refresh Route Handler called by a Vercel cron job, and the supporting config changes (next.config.ts image domains, vercel.json cron schedule).

Purpose: The Instagram Graph API blocks CORS from browsers, so all API calls must be proxied server-side. The long-lived token (60-day TTL) must be refreshed daily via cron to prevent silent expiry. This infrastructure is what the InstagramFeed client component in Plan 03 calls.

Output: Two new Route Handler files, updated next.config.ts, updated vercel.json with cron config.
</objective>

<task type="auto">
  <name>Task 1: Create Instagram feed Route Handler proxy</name>
  <files>src/app/api/instagram/feed/route.ts</files>
  <action>
    Create the directory `src/app/api/instagram/feed/` and the file `route.ts`.

    This Route Handler:
    - Exports a named `GET` function (Next.js App Router Route Handler convention)
    - Sets `export const dynamic = 'force-dynamic'` at the top — do NOT cache the route itself (the upstream fetch uses revalidate for server-side caching)
    - Reads `process.env.INSTAGRAM_ACCESS_TOKEN`; if falsy, immediately returns `Response.json({ posts: [] }, { status: 200 })`
    - Fetches from `https://graph.instagram.com/me/media?fields=id,media_type,media_url,thumbnail_url,permalink,timestamp&limit=9&access_token={token}`
    - Uses `{ next: { revalidate: 3600 } }` on the upstream fetch (1-hour server-side cache — balances freshness with rate limits; NOT cached at the Route Handler level)
    - If the upstream fetch returns `!res.ok`, returns `Response.json({ posts: [] }, { status: 200 })`
    - Filters the `data.data` array to only include entries where `media_type === 'IMAGE' || media_type === 'CAROUSEL_ALBUM'` (VIDEO posts lack `media_url`)
    - Returns `Response.json({ posts })` with the filtered array
    - Wraps the entire fetch block in try/catch — catch returns `Response.json({ posts: [] }, { status: 200 })`
    - NEVER returns a 500 or error status — all failures return `{ posts: [] }` with 200

    TypeScript: define an inline `InstagramPost` interface with fields `id: string`, `media_url: string`, `thumbnail_url?: string`, `permalink: string`, `media_type: 'IMAGE' | 'CAROUSEL_ALBUM'`. Use this to type the posts array before returning.

    Do NOT use `instagram-web-api` or any npm package — use native `fetch` only.
    Do NOT call `https://api.instagram.com` — that is the deprecated Basic Display API (shut down Dec 2024). Only `https://graph.instagram.com`.
  </action>

<task type="auto">
  <name>Task 2: Create token refresh Route Handler and update vercel.json and next.config.ts</name>
  <files>src/app/api/instagram/refresh-token/route.ts, vercel.json, next.config.ts</files>
  <action>
    Three related infrastructure changes — group in one task because they are all small config-level edits:

    **A. Create `src/app/api/instagram/refresh-token/route.ts`:**
    - Create directory `src/app/api/instagram/refresh-token/` and file `route.ts`
    - Exports a named `GET` function
    - Reads `process.env.INSTAGRAM_ACCESS_TOKEN`; if falsy, returns `Response.json({ error: 'No token configured' }, { status: 500 })`
    - Calls `https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token&access_token={token}`
    - If `res.ok` and `data.access_token` exists, returns `Response.json({ refreshed: true, expires_in: data.expires_in })`
    - If refresh fails, returns `Response.json({ refreshed: false, error: 'Refresh failed' }, { status: 500 })`
    - Includes a comment: `// NOTE: The new token is returned in the response but cannot be stored back to Vercel env vars at runtime. As long as this cron runs daily, the existing token's 60-day window is continuously extended. Document this as a known limitation.`
    - Important: This endpoint DOES NOT update `process.env` — it just calls the refresh endpoint to keep the token alive. As long as the token is refreshed daily (before 60 days expire), the same token value in the env var remains valid.

    **B. Update `vercel.json`:**
    - Read the existing vercel.json (has buildCommand and headers)
    - Add a `"crons"` array at the top level:
      ```json
      "crons": [
        {
          "path": "/api/instagram/refresh-token",
          "schedule": "0 3 * * *"
        }
      ]
      ```
    - Preserve all existing keys (buildCommand, headers). Do not remove them.

    **C. Update `next.config.ts`:**
    - Read the existing next.config.ts (currently empty config: `const nextConfig: NextConfig = {}`)
    - Add `images.remotePatterns` to allow Instagram CDN hostnames:
      ```typescript
      const nextConfig: NextConfig = {
        images: {
          remotePatterns: [
            {
              protocol: 'https',
              hostname: '**.cdninstagram.com',
            },
          ],
        },
      };
      ```
    - Preserve the import and export default. Do not change other config keys.
  </action>
