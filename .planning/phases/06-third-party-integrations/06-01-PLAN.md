<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Set up Instagram Graph API credentials</name>
  <action>
    Complete the following steps to obtain a long-lived Instagram access token. These are one-time manual steps with no automated equivalent.

    Step 1 — Verify account type:
    - Open the @playrevolution Instagram app on mobile
    - Go to Settings → Account → Switch to Professional Account (Creator or Business)
    - If already a Professional account, skip this step

    Step 2 — Create a Meta developer app:
    - Go to developers.facebook.com → My Apps → Create App
    - Select "Other" as use case, then "Business" as app type
    - Name the app (e.g. "Playful Revolution Games Index")
    - On the app dashboard, click "Add Product" → find "Instagram" → click "Set up" under "Instagram API with Instagram Login"

    Step 3 — Generate a long-lived token:
    - In the Meta developer app, go to Instagram → API setup with Instagram Login
    - Under "Generate access tokens", connect your @playrevolution Instagram Professional account
    - Copy the short-lived token that appears
    - Exchange it for a long-lived token (60-day TTL) using the Graph API Explorer or with this curl command:
      ```
      curl -X GET "https://graph.instagram.com/access_token?grant_type=ig_exchange_token&client_secret={YOUR_APP_SECRET}&access_token={SHORT_LIVED_TOKEN}"
      ```
    - Copy the `access_token` value from the response — this is the long-lived token

    Step 4 — Add token to Vercel:
    - Go to vercel.com → your project → Settings → Environment Variables
    - Add: Name = `INSTAGRAM_ACCESS_TOKEN`, Value = the long-lived token, Environment = Production + Preview + Development
    - Click Save

    Step 5 — Add token to local .env.local:
    - Open (or create) .env.local in the project root
    - Add: `INSTAGRAM_ACCESS_TOKEN=your_long_lived_token_here`
    - Verify .env.local is in .gitignore (it should be — Next.js adds it by default)
  </action>
  <resume-signal>Type "instagram-done" when INSTAGRAM_ACCESS_TOKEN is set in both Vercel and .env.local</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Set up Sender.net form ID</name>
  <action>
    Complete the following steps to retrieve the Sender.net form ID.

    Step 1 — Create or locate the signup form:
    - Log in to app.sender.net
    - Go to Forms → Create New Form (or use an existing form if one already exists)
    - Set up a simple email signup form with the name "Playful Revolution Newsletter" (or similar)
    - Save/publish the form

    Step 2 — Retrieve the form ID:
    - Open the form → click "Embed" or "Get embed code"
    - Locate the `data-sender-form-id` attribute in the embed snippet — the value is a numeric ID (e.g. `data-sender-form-id="123456"`)
    - Copy the numeric value only

    Step 3 — Add form ID to Vercel:
    - Go to vercel.com → your project → Settings → Environment Variables
    - Add: Name = `NEXT_PUBLIC_SENDER_FORM_ID`, Value = the numeric form ID, Environment = Production + Preview + Development
    - Click Save

    Step 4 — Add form ID to local .env.local:
    - Open .env.local in the project root
    - Add: `NEXT_PUBLIC_SENDER_FORM_ID=your_form_id_here`
  </action>
  <resume-signal>Type "sender-done" when NEXT_PUBLIC_SENDER_FORM_ID is set in both Vercel and .env.local</resume-signal>
</task>
