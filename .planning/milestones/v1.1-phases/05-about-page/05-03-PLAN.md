  <action>
Update `public/admin/config.yml` by appending an `about` file collection entry to the existing `collections` list. Keep all existing `games` collection configuration unchanged.

The new collection uses `files:` (not `folder:`) for the singleton pattern — it points to a single file, not a directory:

```yaml
  - name: "about"
    label: "About Page"
    files:
      - name: "about"
        label: "About Page"
        file: "content/about.md"
        fields:
          - { label: "Organisation Name", name: "organisationName", widget: "string" }
          - { label: "Description (short)", name: "description", widget: "text" }
          - label: "Social Links"
            name: "socialLinks"
            widget: "list"
            fields:
              - label: "Platform"
                name: "platform"
                widget: "select"
                options: ["instagram", "twitter", "facebook", "linkedin", "youtube"]
              - label: "URL"
                name: "url"
                widget: "string"
                pattern: ["^https?://", "Must start with http:// or https://"]
          - { label: "Body", name: "body", widget: "markdown" }
```

Key details:
- Uses `files:` (list) inside the collection — this is Decap CMS's file collection pattern for singletons
- `file: "content/about.md"` must exactly match the path of the seed file created in Plan 01
- Platform `select` widget limits choices to known icon set in SocialLinks.tsx
- URL `pattern` validates that URLs start with http:// or https://
- Field names (`organisationName`, `description`, `socialLinks`, `platform`, `url`, `body`) must exactly match frontmatter keys in content/about.md and field names in src/lib/about.ts
- Do NOT add `create: true` or `delete: true` — file collections manage a fixed file
